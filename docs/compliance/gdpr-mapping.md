# GDPR Principle Mapping (CarePoint)

## Lawfulness, fairness, transparency
- Documented privacy notice (`privacy-notice.md`)
- Defined legal bases for core processing

## Purpose limitation
- Each major dataset has explicit purpose tied to platform functions
- No unrelated secondary use documented

## Data minimization
- Optional fields remain optional where possible
- Avoid duplicate sensitive/plaintext copies where unnecessary

## Accuracy
- User profile update routes available
- Admin correction workflow exists for verification records

## Storage limitation
- Defined retention schedule (`retention-policy.md`)
- Deletion/anonymization approach documented

## Integrity and confidentiality
- HTTPS, secure cookie configuration, role checks, MFA, RLS
- Rate limits and origin validation for mutation routes

## Accountability
- Audit logs and incident procedure documented
- Rights-handling workflow documented
- Vendor/processor register maintained

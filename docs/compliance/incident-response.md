# Incident Response Procedure

## Trigger
Any suspected unauthorized access, data leakage, or integrity compromise involving personal data.

## Steps
1. **Containment**
   - Disable affected integrations/keys/sessions.
   - Restrict impacted routes or services.
2. **Assessment**
   - Determine affected systems, data categories, and user impact.
3. **Documentation**
   - Record timeline, root cause, impact scope, and mitigations.
4. **Notification**
   - Notify supervisory authority if legally required (target within 72 hours for notifiable breaches).
   - Notify affected users without undue delay if high risk exists.
5. **Recovery**
   - Patch vulnerability, rotate credentials, verify clean state.
6. **Post-incident review**
   - Add prevention actions and update controls/policies.
